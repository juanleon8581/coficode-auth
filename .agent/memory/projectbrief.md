# Project Brief

## Main Objective

CofiAuth is a software project originally designed to serve as a base (template or boilerplate) for the agile creation of new projects.

## Problem it Solves

When initiating new developments, the same fundamental steps are frequently repeated:

1. Base dependencies configuration.
2. Authentication system integration.
3. Testing environment configuration.
4. Filesystem structure establishment.

CofiAuth seeks to eliminate this repetition by providing a solid starting point, ready to be cloned.

## Key Requirements

- **High parameterization**: It must be extremely easy to adapt (change logos, names, API keys, etc.) exclusively through the use of environment variables.
- **Efficiency**: Avoid repetitive configuration time to be able to focus directly on the business logic of the new project.
